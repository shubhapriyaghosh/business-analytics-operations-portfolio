# Analytics & Operations Project 2
## Finance Operations Reconciliation & Exception Management System

### Purpose
This project simulates a finance operations reconciliation system that identifies transaction exceptions such as amount mismatches, duplicate references, missing bank records, missing ledger records and status mismatches.

### What is included
- Synthetic transaction-level reconciliation dataset.
- Rule-based exception classification.
- Exception dashboard by type and priority.
- Controls matrix for auditability and operating ownership.
- Action tracker for exception resolution.
- Source sheet with plain-text URLs.

### How to use
Open the Excel workbook and review:
1. `Overview`
2. `Rules`
3. `Transactions`
4. `Exception_Dashboard`
5. `Controls_Matrix`
6. `Action_Tracker`
7. `Sources`

### Privacy
All transaction data is synthetic. No real financial, customer or merchant records are used.
